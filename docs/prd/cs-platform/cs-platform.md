# 客服系统 PRD

> 状态：草稿中，对话采集进行中。

## 1. 背景与目标

### 当前痛点
现有客服系统不好用，主要体现在三个方面：

1. **信息分散**：客服在处理一次咨询时，需要在多个系统之间来回切换才能拼齐客户信息（基础资料、历史订单、过往沟通记录等）。
2. **查找慢**：历史工单和知识库的检索效果差，客服很难快速找到相似案例或标准答案。
3. **协作差**：工单升级、转派、跨部门协同的流程不顺畅，沟通成本高、响应慢。

### 目标
- 让客服在**一个工作台**内就能获取处理一次咨询所需的全部信息。
- 让历史工单和知识库**可被快速、准确地搜索到**。
- 让工单流转、跨部门协作**可视、可追踪、低摩擦**。
