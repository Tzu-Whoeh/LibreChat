# 客服系统 PRD

> 状态：草稿中，对话采集进行中。

## 1. 背景与目标

### 当前痛点
现有客服系统不好用，主要体现在三个方面：

1. **信息分散**：客服处理一次咨询，需要在 CRM（客户资料）、ERP（订单）、邮箱（历史邮件）、知识库（标准答案）之间来回切换。
2. **查找慢**：历史工单和知识库的检索效果差，难以快速找到相似案例或标准答案。
3. **协作差**：工单升���、转派、跨部门协同流程不顺畅，沟通成本高。

### 目标
- 让客服在**一个工作台**内就能获取处理一次咨询所需的全部信息（客户资料 + 订单 + 历史沟通 + 知识库）。
- 让历史工单和知识库**可被快速、准确地搜索到**。
- 让工单流转、跨部门协作**可视、可追踪、低摩擦**。
- 统一接管**电话、邮件、网站在线聊天**三个渠道的咨询。

## 2. 目标用户

- **一线客服**（主要用户）：约 20 人规模的内部团队，处理客户咨询、创建/流转工单。
- _待补充：客服主管 / 跨部门协作方 / 管理者 是否也是本期目标用户？_

## 3. 核心场景

### 渠道接入
客户咨询从 **电话、邮件、网站在线聊天** 三个渠道进入，统一汇聚到客服工作台。

### 一次咨询的处理
客服在工作台内：
1. 接到来自任一渠道的咨询。
2. 系统自动带出客户资料（来自 CRM）、订单信息（来自 ERP）、历史沟通记录（含历史邮件）。
3. 客服根据问题搜索知识库 / 历史工单，找到答案。
4. 必要时升级、转派或拉跨部门同事协作。
5. 结束会话，沉淀工单。

## 5. 功能模块概览

初步方向（随对话补充）：
- **统一工作台**：客户信息卡 + 订单 + 历史沟通 + 知识库 一屏可见。
- **全渠道接入**：电话 / 邮件 / 在线聊天 汇聚为统一会话。
- **全局搜索**：跨工单、知识库、客户资料的搜索。
- **工单协作与流转**：升级、转派、跨部门协同。
