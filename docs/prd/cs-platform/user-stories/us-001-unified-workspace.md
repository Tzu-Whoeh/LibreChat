# US-001 客服在一个工作台看齐客户全貌

## 角色
一线客服

## 故事
**作为** 一名一线客服，
**我希望** 在接到一通客户咨询时，工作台能自动带出该客户的资料、订单、过往沟通记录，
**以便** 我不必在 CRM、ERP、邮箱之间反复切换，能立刻进入问题本身，提高响应速度和服务质量。

## 触发场景
- 电话呼入并通过号码识别到客户。
- 客户从网站在线聊天发起会话（已登录或通过邮箱识别）。
- 收到客户邮件并匹配到已有客户。

## 期望结果
- 客服打开会话的同时，**客户信息卡片**自动加载到工作台一侧。
- 卡片至少包含：
  - 基础资料（来自 CRM）
  - 最近的订单（来自 ERP）
  - 过往沟通记录（含历史邮件、历史工单、历史会话）
- 全部数据在客服开始打字前就已就绪，不需要手动搜索。

## 待澄清
- 客户识别失败（陌生号码 / 未注册邮箱）时如何兜底？
- 客户资料字段优先级？哪些必须一眼看到？
- 订单要展示几条、按什么排序？
