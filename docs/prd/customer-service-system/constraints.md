# 业务约束

> 用户能感受到的要求，由架构 agent 翻译成技术方案。

## 规模与性能
- 客服并发：约 20 人同时在线
- 消息送达延迟：≤ 3 秒
- AI 推荐答案生成：≤ 5 秒
- 录音转文字处理：单段录音（< 30 分钟）≤ 30 秒返回结果
- 周报生成：周一 09:00 准时推送，可接受 ±5 分钟误差

## 数据集成
- **知识库**：系统内自维护
- **企业文档库**：SharePoint，通过 API 检索；需保证权限隔离（客服只能看到自己权限内的文档）
- **微信渠道**：具体是企业微信还是公众号客服待确认
- **电话**：仅事后录音上传，不接入实时通话系统
- **邮件**：用于推送周报，需要企业邮件服务（SMTP 或集成 Outlook）

## 访问范围
- 系统**仅供公司内部员工**使用
- **客服**：只能看到自己处理的会话和全量知识库；不能管理知识库与敏感规则
- **客服主管**：可见全部会话、可管理知识库、可管理敏感规则、可查看周报与看板
- **相关团队**：不开放系统账号
- 终端用户（被服务的客户）：只在网页/微信端与客服交互，不接触系统后台

## 业务规则
- **AI 必须人工把关**：AI 不允许在未经客服确认的情况下直接发送消息给用户
- **敏感问题强制人工**：命中敏感规则时 AI 不出答，必须客服人工处理
- **优质对话沉淀**：CSAT ≥ 4 星自动进入自学索引；客服也可主动标记
- **周报闭环**：相关团队不登录系统，由主管线下分发

## 数据安全与合规
- 用户对话内容、录音、转写文本属于敏感数据，需在传输和存储中加密
- 录音文件保留周期待定，建议初版 6 个月
- 客服访问对话需有审计日志（谁、何时、看了什么）
- 满足国内个人信息保护相关法规要求

## 可用性
- 工作日 09:00–21:00 为核心使用时段，期间可用性 ≥ 99.5%
- 非核心时段允许维护窗口
- 单点故障不允许造成对话消息丢失

## 待补充
- CSAT 与响应时长的基线数据（上线前需采集）
- SharePoint 具体站点与文档范围
- 微信渠道类型（企业微信 / 公众号客服）
- 录音文件保留周期
- 是否有等保、GDPR 等额外合规要求
